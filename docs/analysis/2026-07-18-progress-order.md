# Анализ порядка блоков раздела «Моя игра» (2026-07-18)

## Промт (переиспользуемый)

Ты — продуктовый дизайнер мобильных habit/фитнес-трекеров (Strava, Duolingo,
Whoop-класс). Проанализируй порядок вывода информации на главном экране
статистики приложения-дневника BJJ для белых поясов.

Правила анализа:
1. У экрана три джобы в порядке убывания ценности: «что мне делать дальше»
   (действие), «как мои дела» (статус), «кто я как игрок» (идентичность).
2. Блоки-действия выше блоков-справки. Частота обновления данных блока
   коррелирует с высотой: живое сверху, медленное вниз.
3. Проверяй два персонажа: новичок (0-2 записи, порог архетипа не пройден)
   и активный (дневник ведётся, 15+ техник). Порядок должен работать для обоих
   без пустых дыр в первом экране.
4. Дофаминовые якоря (стрик, прогресс, шеринг) не прячутся ниже третьего экрана.
5. Смысловые пары не разрывать (статус -> вытекающее действие).
6. Предлагай минимальную перестановку, не редизайн. Отдельно: что отдать
   телеметрии, чего не делать.

Вход: список блоков сверху вниз с их содержимым, условиями показа и типом
(действие/статус/идентичность/справка).

Выход: таблица блоков с джобами, диагнозы, целевой порядок с обоснованием
каждого перемещения, открытые вопросы к телеметрии.

## Вход: текущий порядок (после п.18.3-18.5)

1. Hero 2x2: Прогресс % (шторка пояса+группы) / Профиль (лист игрока) /
   Изучено (инлайн-список) / В процессе (инлайн-список) — статус, всегда.
2. Текущий фокус + Следующая цель — статус + действие, всегда.
3. Что тебя ловит — действие (защиты от повторных сабмишенов), при 2+ поимках.
4. Пора повторить — действие (стёкшее изученное), при ведущемся дневнике.
5. Твой стиль — идентичность (квест «N из 5» до порога; архетип + шеринг после).
6. Характеристики (6 статов) — справка, всегда.
7. Разрыв (GapCard) — идентичность + действие («ближайшие 3 техники стиля,
   собрать тренировку»), при заданной аспирации и пройденном пороге.

## Диагнозы

1. ГЛАВНОЕ: «Разрыв» — по спеку геймификации ядро ценности раздела и
   единственный блок идентичности с прямым действием — стоит ПОСЛЕДНИМ, ниже
   чисто справочных статов. Активный пользователь с аспирацией доскролливает
   до него в последнюю очередь.
2. Разорвана смысловая пара «Твой стиль» (кто я) -> «Разрыв» (кем хочу быть и
   что для этого делать): между ними вклиниваются статы-справка.
3. Верх экрана здоров: hero-статус, затем действия (цель, ловит, повторить)
   в правильном порядке остроты (ловит = боль, выше повторить = гигиена).
4. Для новичка порядок тоже работает: ловит/повторить/разрыв скрыты, квест
   «N из 5» виден на первом-втором экране, статы нулевые в конце не мешают.
5. Дубли смыслов в hero (Прогресс % vs Изучено N/293) — кандидат на упрощение,
   но это отдельное решение после телеметрии, не порядок.

## Целевой порядок (минимальная перестановка: одна)

1. Hero — как есть.
2. Фокус + Цель — как есть.
3. Что тебя ловит — как есть.
4. Пора повторить — как есть.
5. Твой стиль — как есть.
6. РАЗРЫВ — поднять сюда (смысловая пара со стилем, действенный блок).
7. Характеристики (6 статов) — вниз (чистая справка, самая медленная).

## Телеметрии на подтверждение

- reco_click detail=gap до/после перестановки (вырастет — гипотеза верна).
- Если section_open(progress) высокий, а reco_click(gap) около нуля и после
  перестановки — аспирацию мало кто задаёт; тогда думать про онбординг
  аспирации, а не про порядок.
- Объединение трёх блоков действий (цель/ловит/повторить) в один «Что делать
  сегодня» — только по данным reco_click (уже в дальнем бэклоге).

## Чего не делать

- Не дублировать квест стиля бейджем в hero (лишняя сущность против упрощения).
- Не трогать hero-карточки до телеметрии.
- Не сворачивать статы в шторку: они лёгкие и внизу никому не мешают.
