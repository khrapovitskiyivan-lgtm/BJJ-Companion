# UX-разбор: дружелюбность и удобство использования

Роль: UX-специалист по ИА и когнитивной нагрузке (персона `.claude/agents/bjj-ux-reviewer.md`), расширено на общую дружелюбность (первое впечатление, навигация, понятность «что тут делать», обратная связь, эргономика).
Дата: 2026-07-23. Метод: живой проход экранов в мобильном вьюпорте (375x812) на dev-сборке, DOM-осмотр (скриншоты превью таймаутят — вероятно intro.mp4 на первом кадре). Состояние: свежий первый запуск + населённый вернувшийся пользователь (дневник за 2 недели, прогресс, «ловит»).

> Не правка кода — разбор с вариантами. Цель владельца: рост+удержание, не выхолащивая петлю дневник->стат/стиль->рекомендации.

## Резюме (5 строк)

Ядро дружелюбное: чистая нижняя навигация (4 пункта), понятные одноцелевые экраны, консистентный дизайн (navy/золото, Inter, скруглённые карточки), честные пустые состояния, прогрессивное раскрытие в форме дневника. Три реальных трения: (1) **первое впечатление холодное** — до ценности два юридических экрана + 7-шаговый онбординг; (2) **хаб «Моя игра» перегружен** — 9 блоков, ~2.9 экрана, теперь ЧЕТЫРЕ конкурирующих блока «что делать дальше» (мой свежий «Разбери показанное» добавил четвёртый) и трижды повторённый CTA «Собрать отработку»; (3) **«Техники» открываются на графе** — самое сложное представление первым для аудитории белых поясов.

## Проблема 1 — Хаб «Моя игра»: слабая иерархия + дубли действий (🔴)

Факт (населённое состояние): 9 секций сверху вниз — Сегодня · **Разбери показанное** · Партнёры · Прогресс(hero) · **Текущий фокус/Следующая цель** · **Что тебя ловит** · **Пора повторить** · Твой стиль · Характеристики(свёрнуто). Все примерно одного визуального веса.

Два конкретных источника трения:
- **Четыре блока «что делать дальше» подряд** (выделены жирным). Прошлый разбор (2026-07-22) отмечал ТРИ; мой блок «Разбери показанное» этой сессии добавил четвёртый. Пользователю четыре разных «вот что дальше» = паралич выбора (закон Хика).
- **CTA «Собрать отработку по дневнику» повторяется 3 раза** (в «Что тебя ловит», «Пора повторить» + «В отработку» в «Разбери показанное») — визуальный шум, три кнопки в одну цель.

Варианты:
- **Оставить** — если телеметрия (`reco_click` по detail + свежие `review_opened`/`review_drill`) покажет, что кликают по всем. Усилие 0. Риск: копится плотность.
- **Дедуп CTA** (S): один общий «Собрать отработку» внизу группы рекомендаций вместо трёх. Дешёвый выигрыш, ценность не теряется.
- **Свернуть второстепенные реко в аккордеон** (S-M): открыт «Разбери показанное» (свежак после тренировки) ИЛИ «Текущий фокус»; «Что тебя ловит»/«Пора повторить» — под «Показать ещё». 
- **Объединить в один блок «Что дальше»** с 3 строками (по одной каждого типа) + «ещё» (M). CLAUDE.md держал это «только после телеметрии» — данные теперь есть.
- **Рекомендация:** сначала дедуп CTA (S) + свернуть «Пора повторить»/«Что тебя ловит» (S). «Разбери показанное» оставить видимым (это крючок удержания и он само-очищается). Радикальное объединение — по `reco_click`/`review_*`, не на глаз.

Заметка честности: retention-работа этой сессии реально повысила плотность хаба, которую прошлый разбор уже помечал 🔴. Это аргумент заняться иерархией «Моей игры» раньше в очереди, чем откладывалось.

## Проблема 2 — «Техники» открываются на графе (🟡)

Факт: тап «Техники» -> `/map`, первая вкладка «Карта» (граф React Flow), кикер «Технический атлас». Подвкладки: Карта · Список · Что если · Словарь. Белопоясник (целевая аудитория) сразу видит самое сложное представление; дружелюбный «Список» (библиотека с фильтрами) — вторым. Совпадает с опросом (граф = «карта метро» для новичков, п.25.4).

Варианты:
- **Оставить** (0) — граф как «вау-фича» первым; но опрос против для новичков.
- **Дефолтная вкладка по поясу/прогрессу** (S-M): белый/синий или мало `done` -> открывать «Список»; выше -> «Карта». Наименьшее удивление для новичка.
- **Поменять порядок вкладок** (S): «Список» первым для всех, «Карта» второй. Проще, но лишает опытных «вау».
- **Рекомендация:** дефолт «Список» для белых поясов (или при `countDone` ниже порога), «Карта» для остальных. Граф не убираем (решение CLAUDE.md — оставить), просто не встречаем им новичка.

## Проблема 3 — Холодное первое впечатление (🟡)

Факт: новый пользователь видит подряд (1) экран согласия с плотным юр-текстом про обработку данных и трансграничную передачу, (2) экран «Локальный режим» с таблицей что работает/не работает, и только (3) онбординг с карточками ценности («Дневник и план», «310 приёмов», «Умная отработка»). То есть **два юридических экрана до первой ценности**. Согласие-первым мотивировано 152-ФЗ (осознанное решение CLAUDE.md), но UX-цена — холодный старт: просят согласие раньше, чем показали, зачем оно.

Варианты:
- **Оставить** (0) — юридически чище (согласие до любой обработки). Помечаю как приоритет легальности над UX — решение владельца.
- **Один экран-тизер ценности ПЕРЕД согласием** (S-M): 1 слайд «что это и зачем» -> потом гейт. Согласие остаётся до обработки, но пользователь понимает контекст. Не нарушает 152-ФЗ (тизер данных не обрабатывает).
- **Свернуть «Локальный режим» в тултип/аккордеон** на гейте (S): не отдельный полноэкранный шаг, а «Использовать локально -> короткое подтверждение».
- **Рекомендация:** добавить 1 тизер-слайд до гейта (S-M) — самый дешёвый способ согреть первое впечатление, легально безопасен. «Локальный режим» ужать до подтверждения (S).

Смежное: intro.mp4 (1 МБ) грузится на welcome-слайде первым кадром (скриншоты превью на нём таймаутят). На медленном мобиле это медленный первый экран у нового пользователя (SW кэширует лишь потом, п.18.8). Кандидат: постер-кадр + ленивая загрузка видео, чтобы welcome красился мгновенно.

## Проблема 4 — «310» в статичных текстах (🟢 тривиально, но это баг) — СДЕЛАНО 2026-07-23

Онбординг: «310 приёмов и позиций». Hero «Моей игры» считает верно: «7 / 315». «310» захардкожено в `Onboarding.tsx` и двух сообщениях бота (`api.tg-webhook.ts`). В `/about` хардкода «310» НЕТ (проверено). Исправлено на «315».

## Не трогать — здорово

- Нижняя навигация: 4 пункта (Моя игра · Дневник · Техники · Отработка), ясные подписи+иконки.
- Форма дневника: дата -> чипы выбранного -> поиск -> чипы групп -> вторичное под «Подробнее». Образцовое прогрессивное раскрытие.
- Дизайн-система: единый navy/золото, Inter, скруглённые карточки, кикер 11px uppercase, консистентно на всех экранах.
- Честные пустые/нет-данных состояния («Видео-разбора пока нет / Нужен видео-разбор», «Прямых защит в базе нет — разбери позицию на карте»).
- Прогрессивное раскрытие уже применено: Характеристики свёрнуты, «Кто в игре» аккордеоном, вторичные поля дневника под «Подробнее».
- Кнопки «Назад» заметны (заливка primary), карточка техники линейна и сканируема.

## Приоритетные ходы (по цене/эффекту)

1. **«310» -> «315»** в онбординге и сообщениях бота (S) — СДЕЛАНО 2026-07-23 (в `/about` хардкода не было).
2. **Дедуп CTA «отработка» на «Моей игре»** — ЧАСТИЧНО СДЕЛАНО 2026-07-23: убран один из двух идентичных соседних футер-CTA (в «Что тебя ловит»; «Пора повторить» сохраняет). Полное слияние CTA через блоки/GapCard + ReviewShownBlock — это переделка хаба, вынесено к п.5/доп.эксперту (не соло по хабу).
3. **Дефолт «Список» в «Техниках» для белых поясов** (S-M) — не встречать новичка графом.
4. **Тизер-слайд до гейта согласия + ужать «Локальный режим»** (S-M) — согреть холодный старт, легально безопасно.
5. **Иерархия «Моей игры»** (M): свернуть «Что тебя ловит»/«Пора повторить» под «ещё»; объединение трёх реко-блоков — решить по `reco_click` + `review_*` (телеметрия уже есть). Заняться раньше, чем откладывалось: retention-блок этой сессии повысил плотность.
6. (Смежно, перф) постер-кадр + ленивое intro.mp4 на welcome (S-M) — быстрый первый экран.
