Роль: продакт по спорт/habit-companion-приложениям (персона `.claude/agents/bjj-product-reviewer.md`).
Дата: 2026-07-22. Опора: код + первый аудит `2026-07-22-product-audit.md`.
Фокус сессии: что изменилось — гейт согласия перед онбордингом и «Партнёры по залу» (Шаг Б).

## Резюме (что сильно / что слабо)

Ядро по-прежнему сильное: петля дневник -> стат/стиль/рекомендации, база 310, «Сегодня». Партнёры — правильная ставка на удержание и вирусность, реализованы приватно (наружу только агрегаты) и безопасно (проверка подписи Telegram). НО появились два новых риска на краях воронки, оба бьют по заявленной цели РОСТА: (1) гейт согласия встал ПЕРВЫМ экраном, до показа ценности — риск просадки активации; (2) главная ростовая фича выкачена БЕЗ телеметрии — мерить воронку приглашений нечем, летим вслепую. Плюс у партнёров классический холодный старт (нужны двое, малая база, только Telegram).

## Топ-5 приоритетов (проблема -> изменение -> эффект -> усилия)

1. **Телеметрия партнёров и гейта (СНАЧАЛА).** Проблема: главная ростовая фича и новый барьер активации не измеряются — `partners.ts`/`ConsentGate.tsx` не шлют событий; в белом списке `telemetry.ts` их нет. Изменение: добавить события `consent_accept`/`consent_local` (видеть, сколько отваливается на гейте) и `invite_created`/`invite_accepted`/`partner_opened` (воронка приглашений). Эффект: видно, работает ли ставка на рост и во что обходится гейт. Усилия: S. Без этого все решения ниже — гадание.
2. **Пересмотреть место гейта согласия: «ценность вперёд, согласие по требованию».** Проблема: юридически мотивированный гейт стоит первым экраном, до онбординга и до любой ценности; текст «юридический», почта-заглушка — плохое первое касание для роста. Изменение: пустить в онбординг+локальный режим сразу, а согласие спрашивать в момент, когда пользователь впервые тянется к серверной функции (партнёры/синхронизация/статистика). Эффект: активация не страдает, согласие — в контексте (сильнее и по смыслу). Усилия: M. Конфликт: юрист может хотеть гейт раньше — решение за пользователем/юристом.
3. **Холодный старт партнёров.** Проблема: ценность фичи = когда твои партнёры по залу тоже в приложении; при малой базе у большинства будет пустой список. Изменение: делать инвайт-момент ценным даже с 0 партнёров (промпт «позови партнёра по залу» после первой недели в плане/серии), максимально короткий инвайт, возможно QR/ссылка «поделиться в чат зала». Эффект: запуск петли не упирается в пустоту. Усилия: M.
4. **Длина главного экрана / когнитивная нагрузка.** Проблема: «Моя игра» растёт (Сегодня -> Партнёры -> hero -> цель -> ловит -> повторить -> стиль -> разрыв -> характеристики -> группы). Изменение: следить за приоритетом; при пустом списке партнёров держать блок компактным (уже так), не давать ему разрастаться. Эффект: верх экрана остаётся читаемым. Усилия: S. Не резать до данных телеметрии.
5. **Закрыть юр-хвосты перед активной раскруткой.** Проблема: в проде черновик Политики/Условий, почта-заглушка, вопрос 242-ФЗ не решён, имена/фото граждан РФ на сервере за границей — риск растёт с числом пользователей. Изменение: до платного/массового привлечения — почта, консультация юриста, решение по локализации. Эффект: рост без юридической мины. Усилия: внешнее.

## Уже хорошо (не переделывать)

- Ядро-петля и «Сегодня» — не трогать.
- Партнёры сделаны приватно правильно: наружу только стиль/характеристики/статус недели, без дневника/заметок/техник; выбор пользователя тут здоровый.
- Безопасность партнёров: проверка подписи Telegram на сервере, RPC только под service_role — подделать чужой id нельзя.
- Инвайт по личному deep-link с кодом (не device_id), приём — явной кнопкой.
- Согласие реально гейтит отправку (проверено: 0 запросов до согласия), есть локальный режим и отзыв.

## Открытые вопросы

1. Рост через зал (зову реальных партнёров по залу) или широкий? От этого зависит решение холодного старта (п.3).
2. Готов ли перенести согласие на «по требованию» ради активации, или юрист требует гейт первым? (п.2 — конфликт активации и юр-осторожности).
3. Когда включать телеметрию партнёров — сейчас (рекомендую, чтобы первые же инвайты мерились) или после стабилизации фичи?
