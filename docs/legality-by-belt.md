# Легальность по поясу (2026-07-20)

Фича: бейджи легальности в карточке техники учитывают пояс пользователя из профиля.

## Механизм
Колонки `legal_ibjjf_gi` / `legal_ibjjf_nogi` / `legal_adcc` в `data/techniques.csv` теперь принимают:
- `✔` — легально всем поясам,
- `✗` — не применимо/запрещено (бейдж не показывается, как раньше),
- слаг пояса (`blue`/`purple`/`brown`/…) — легально С этого пояса.

Парсинг: `scripts/build-data.mjs::legalVal`. Тип: `boolean | Belt` (`src/lib/bjj/types.ts`). Логика статуса: `src/lib/bjj/legal.ts` (`legalStatus`, `legalLabel`). Отрисовка: `LegalBadge` в `src/routes/technique.$id.tsx`.

Поведение бейджа:
- пояс ≥ порога → нейтральный «IBJJF Gi»;
- пояс < порога → янтарный «IBJJF Gi · с коричневого» (порог виден, приём не скрыт);
- `✗` → бейдж скрыт (не отличаем «запрещённый сабмишен» от «неприменимо в этом формате» — данных нет).

`recommend.ts` не менялся: строка-пояс truthy, поэтому в скоринге «соревнования» ограниченный по поясу приём считается легальным (он и так фильтруется по поясу в кандидатах), а `✗` — нет. ADCC у всех `✔`.
