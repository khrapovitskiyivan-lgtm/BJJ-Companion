# Тепловая карта дневника от плановой частоты — дизайн

Дата: 2026-07-16. Статус: утверждён пользователем (вид карты, модель пропусков, поведение без частоты — выбраны из предложенных вариантов).

## Задача

Тепловая карта дневника строится от плановой частоты из профиля (frequency 1/3/4 в неделю), показывает пропуски и сверхплановые тренировки, даёт итог месяца с формулировкой (выполнил / перевыполнил / не выполнил). Смена частоты — в листе игрока.

## Утверждённые решения

1. Вид карты: календарь месяца с навигацией по месяцам (вместо ленты 18 недель).
2. Пропуски: недобор закрытой недели, не разметка конкретных дней. Показываются пунктирными пустыми слотами у недели.
3. Частота не задана: план не считается, календарь без квот и итога, кнопка «Указать частоту» открывает лист игрока.

## Модель плана

- Тренировочный день = день, за который есть хотя бы одна запись дневника (несколько записей в день считаются одним днём).
- Недельная квота N = profile.frequency (тип 1|2|3|4; онбординг предлагает 1/3/4).
- Недели календарные, Пн–Вс. Квота недели считается по календарной неделе целиком, включая дни соседних месяцев на стыке.
- Статус недели:
  - Закрытая (вся в прошлом): тренировочных дней меньше N — недобор (пропуски = N − сделано); дни сверх N (хронологически N+1-й и дальше) — сверхплановые.
  - Текущая неделя: показывается прогресс «сделано/N» без недобора (неделя ещё не закрыта).
  - Будущие недели: без статуса.
- План месяца = round(N × дней_в_месяце / 7). Не зависит от разбиения на недельные строки.
- Итог закрытого месяца: сделано < плана — «План не выполнен», равно — «План выполнен», больше — «План перевыполнен».
- Текущий месяц: план-на-сегодня = round(N × прошедших_дней / 7); сделано >= плана-на-сегодня — «Идёшь по плану», иначе — «Отстаёшь от плана». Плюс «сделано X из Y».
- «Сегодня» передаётся в функции параметром — логика детерминирована и тестируема.

## Файлы

- `src/lib/bjj/plan.ts` — новая чистая логика: сетка недель месяца, тренировочные дни, статусы недель, план и итог месяца.
- `src/lib/bjj/plan.test.ts` — vitest-тесты (рядом с recommend.test.ts, stats.test.ts).
- `src/components/bjj/ActivityHeatmap.tsx` — переработка в календарь месяца; имя компонента и файл сохраняются.
- `src/routes/diary.tsx` — прокинуть frequency из useProfile, состояние выбранного месяца, локальный mount CharacterSheet для кнопки «Указать частоту».
- `src/components/bjj/CharacterSheet.tsx` — четвёртая секция «Частота тренировок» (варианты онбординга: «1-2 раза», «3 раза», «4+ раз» → 1/3/4).

## UI календаря

- Шапка: месяц + год, стрелки навигации (вперёд не дальше текущего месяца), стрик с огоньком остаётся.
- Сетка: 7 колонок Пн–Вс, строки — недели месяца; дни чужого месяца затемнены. Тренировочный день — заливка интенсивностью по числу техник (как раньше), сверхплановый день — золотая метка, сегодня — обводка.
- Справа у каждой недели счётчик «2/3»; у закрытых недель недобор — пунктирные слоты, перевыполнение — «+1» золотом.
- Внизу — строка итога месяца с формулировкой.

## Тесты (ключевые случаи)

- monthPlan: 31 день при N=3 → 13; 28 дней при N=1 → 4.
- Закрытая неделя 2/3 → недобор 1; 4 дня при N=3 → один сверхплановый (хронологически последний).
- Текущая неделя — прогресс, без недобора.
- Итог месяца: все три формулировки + текущий месяц по плану/отстаёт.
- Неделя на стыке месяцев: квота учитывает тренировки обоих месяцев.
