# Дизайн: избранное техник + компактный верх «Моей игры»

Дата: 2026-07-23
Статус: согласован (брейншторм), ждёт плана реализации.
Связано: usability-отчёт `docs/analysis/2026-07-23-usability-friendliness-review.md` (пункты C, D, #5).

## Контекст

Верх «Моей игры» разросся: полноширинный блок «Сегодня» + ряд из 4 плиток (Прогресс, Профиль, Изучено, В процессе), а ниже ещё много блоков (usability-разбор: перегруз). Владелец хочет (1) добавить избранное техник и (2) сжать верх: профиль + статы + новая «Избранное» — компактно, «Сегодня» — рядом, не во всю ширину.

Дизайн покрывает две связанные части: механику избранного (C) и переработку верха (D). Шипятся вместе.

## Часть C. Избранное техник

### C1. Store `useFavorites`

Новый стор в `src/lib/bjj/store.ts`, схема как у `useNotes` (общая шина + localStorage + облако):
- Ключ localStorage `bjj.favorites.v1`, тип `Record<number, true>` (id техники -> отмечена). Массив/Set сериализуется как объект-словарь для простоты merge.
- Облако: колонка `favorites_data jsonb` в `bjj_progress` (как `notes_data`), чтобы избранное не терялось при смене устройства и не толкалось с progress/notes. Синхронизация best-effort с гейтом `hasConsent()` (в локальном режиме не выгружается).
- API: `{ favorites, toggleFavorite(id), isFavorite(id), hydrated }`. `toggleFavorite` добавляет/удаляет ключ, пишет localStorage, синхронизирует облако, публикует снимок.
- SQL-миграция `docs/sql/2026-07-23-favorites.sql`: `alter table bjj_progress add column if not exists favorites_data jsonb;` — применяет пользователь.

### C2. Звезда на карточке техники

В `src/routes/technique.$id.tsx`, верхний ряд действий (сейчас «Назад» слева, «Поделиться» справа) — добавить кнопку-звезду рядом с «Поделиться»:
- Иконка `Star` (lucide); заполнённая (`fill`) + цвет `--brand-gold-ink`, когда в избранном; контур `text-muted-foreground`, когда нет.
- Тап -> `toggleFavorite(tech.id)` + `haptic()`. `aria-label` «В избранное» / «Убрать из избранного».
- Телеметрия: событие `favorite_toggle` (detail = id) — расширить whitelist (см. Общее).

### C3. Просмотр избранного

Избранное открывается тапом по стату «Избранное» в компактном верхе (см. D). Механизм — инлайн-раскрытие списка под верхним блоком, как уже работает у «Изучено»/«В процессе» (стейт `openList` в progress.tsx):
- Расширить `openList` значением `"favorites"`; список — техники из `favorites`, отсортированные (в процессе/изученные как в других списках, либо по группе).
- Пустое состояние: `EmptyState` «Пока пусто» + хинт «Отмечай техники звездой на карточке — они появятся здесь».
- Строки — `TechniqueRow` (ссылка на карточку), как в других раскрытиях.

## Часть D. Компактный верх «Моей игры» (Вариант C)

Заменяет нынешние два элемента: полноширинный `<TodayCard />` и `<section>` с 4 плитками hero. Новый блок — **в самом верху** страницы (сразу под `PageHeader`, выше `ReviewShownBlock` и `PartnersBlock`).

### D1. Раскладка (две колонки)

`grid grid-cols-2 gap-3` (обе колонки равной ширины, `items-stretch`):

**Левая колонка — карточка «Профиль + статы»:**
- Сверху: аватар (кружок с кольцом цвета пояса, имя из Telegram или инициалы, фоллбэк — цветной кружок) + имя (`profile.name || "Боец"`) + подпись «`BELT_LABEL` пояс». Тап по всей строке профиля -> `CharacterSheet` (как сейчас).
- Ниже: 4 стата **списком с тонкими разделителями** (НЕ отдельными плитками — решение владельца): иконка + подпись слева, значение справа. По строке:
  - Прогресс `{pct}%` (иконка TrendingUp, значение цветом `--color-primary`) -> тап открывает `ProgressSheet`.
  - Изучено `{done}` (BookOpen, `--status-done`) -> тап раскрывает список `done` (`openList`).
  - В процессе `{inProgress}` (CircleDot, `--status-progress`) -> тап раскрывает список `in_progress`.
  - Избранное `{favCount}` (Star, `--brand-gold-ink`) -> тап раскрывает список `favorites`.
- Каждая строка — кликабельна (кнопка), с ясной зоной нажатия (min ~36px высоты).

**Правая колонка — карточка «Сегодня»** (рамка `--color-primary`, `flex flex-col`):
- Кикер «Сегодня».
- С частотой: `{done} из {quota}` крупно + «на этой неделе»; тонкая сегмент-полоска (изученная часть `--status-progress`, сверхплановые `--brand-gold`); строка «До плана X, осталось Y дн.» мелко. При закрытом плане — «План недели закрыт» (+«N сверх плана» золотом). Серия недель (`weeksStreak>=2`) — золотой огонёк.
- Без частоты: дневной стрик («N дней подряд») вместо «N из Q».
- Кнопка «Записать» (primary, `mt-auto`) -> `/diary?add`; показывается только если сегодня не записано (`!loggedToday`), иначе — галочка «записано».
- Логика данных — существующая `todayCardModel(entries, frequency, today, trainingDays)`; переносим рендер из `TodayCard.tsx` в узкую колонку (текст ужать под ширину).

### D2. Что переносится и что удаляется

- `TodayCard.tsx` — логику модели оставляем, но рендер встраиваем в правую колонку нового блока (либо рефакторим `TodayCard` в компактный вид, принимающий готовую модель). Решение реализации: вынести новый компонент `ProgressTop.tsx` (двухколоночный верх), внутри — левая колонка (профиль+статы) и правая (Сегодня, на базе `todayCardModel`). Старый `TodayCard` удаляем из `progress.tsx` (или он остаётся только если используется где-то ещё — проверить; сейчас только в progress.tsx).
- Старая `<section grid-cols-2 sm:grid-cols-4>` с 4 плитками (StatCard + профиль-кнопка) — удаляется, её содержимое переезжает в левую колонку.
- `StatCard` — если больше нигде не используется, удалить; иначе оставить.
- Инлайн-раскрытие `openList` (списки done/in_progress) — остаётся, рендерится под новым верхом; добавляется ветка `favorites`.
- `ProgressSheet`, `CharacterSheet` — без изменений, те же триггеры (Прогресс-стат и профиль-строка).

### D3. Новый порядок «Моей игры»

`PageHeader -> ProgressTop (две колонки) -> [раскрытый список openList] -> ReviewShownBlock -> PartnersBlock -> рекомендации (Текущий фокус/Следующая цель, Что тебя ловит, Пора повторить) -> Твой стиль -> Разрыв -> Характеристики`.

## Общее

### Крайние случаи
- Нет имени/фото -> цветной кружок пояса (как сейчас).
- Длинные имена -> `truncate`.
- После записи -> правая колонка показывает «записано» (галочка), кнопка скрыта; высота колонок может разойтись — допустимо (`items-start` или пусть левая задаёт высоту).
- Избранное пусто -> стат «Избранное» = 0; тап раскрывает EmptyState.
- SSR: `new Date()` и сторы только после гидратации (гарды как в `TodayCard`/`ReviewShownBlock`).

### Телеметрия
Новое событие `favorite_toggle` (detail=id). Расширить union `telemetry.ts` + whitelist `bjj_track` (`docs/sql/2026-07-23-favorites.sql` включает и колонку, и обновление функции, либо отдельный create-or-replace). Применяет пользователь.

### Правила проекта
Без эмодзи в исходнике / без em-dash; правило языка цвета (золото — достижения/избранное-тёплое, статусы серый->progress->done); общие контролы `ui.tsx`, строки — `TechniqueRow`; хирургические правки; проверка рантаймом в обеих темах (скриншоты превью таймаутят — проверять DOM через `javascript_tool`).

### Затрагиваемые файлы
Новые: `src/components/bjj/ProgressTop.tsx`, `docs/sql/2026-07-23-favorites.sql`.
Правки: `store.ts` (`useFavorites` + облачная синхронизация favorites_data), `technique.$id.tsx` (звезда), `progress.tsx` (замена верха на `ProgressTop`, ветка `openList="favorites"`), `telemetry.ts` (`favorite_toggle`). Возможное удаление: `TodayCard.tsx` (если не нужен вне progress), `StatCard` (если больше не используется).

### Тесты
- `useFavorites` — как `useNotes`, юнит-покрытие store необязательно (UI-хук); поведение проверяется рантаймом.
- Рантайм: звезда переключает избранное и пишется; стат «Избранное» считает; тап раскрывает список/пустое состояние; компактный верх рендерит обе колонки; «Записать» скрывается после записи; обе темы.
