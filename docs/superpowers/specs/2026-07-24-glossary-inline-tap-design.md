# Спек: тап по терминам словаря в описании техники

Дата: 2026-07-24
Статус: черновик на согласование (дизайн утверждён в диалоге + макет)

## Цель

В описании техники термины из словаря становятся тап-активными: тап открывает
короткую справку (поповер), а термины, у которых есть СВОЯ карточка техники, ведут
на неё. Убирает разрыв «читаю разбор, но не знаю слова» без ухода в отдельный
словарь. Ценность фичи = покрытие словаря, поэтому в этот же заход словарь
расширяется концептами (Давление, Угол, Дистанция и др.).

## Решения (утверждены)

- **Вид справки:** поповер у слова (не шторка). Тап мимо — закрывается.
- **Где подсвечивать:** `concept` + `mechanics` + `keyPoints` (объясняющие блоки).
- **Сколько раз:** только ПЕРВОЕ вхождение каждого термина в блоке (иначе шум).
- **Не подсвечивать** термин, совпадающий с названием текущей техники.
- **Две категории тап-целей:**
  - **Концепт** (есть в словаре) -> поповер с определением.
  - **Название техники** (есть карточка) -> ссылка на `/technique/$id` (полная инфа,
    не дублируем в словаре).
- **Подсветка:** пунктирное подчёркивание + приглушённый цвет (сигнал «тап», текст
  остаётся читаемым).

## Расширение словаря (Тир 1 + Тир 2)

Новая категория `principles` («Принципы») для концептов + существующая `movement`.

Тир 1 — концепты (частота в описаниях: Давление 160, Угол 126, Дистанция 58):
- Давление, Угол, Дистанция, Уровень (смена уровня).

Тир 2 — движения (своих карточек нет):
- Маятник, Инверт (инвертирование), Гранби, Ситаут.

Определения пишет реализация в тоне словаря (коротко, для белого пояса). Термины с
готовой карточкой (Беримболо, Локдаун, Твистер, S-маунт, K-гард, Слайсер,
Раббер-гард) в словарь НЕ добавляем — они линкуются на карточку.

## Морфология матчинга (ядро, главный риск)

Термины в словаре в начальной форме; в прозе склоняются. Нужен матч по основе.

Модель:
- **Основа (stem):** термин с отброшенной финальной гласной/ь (а/я/о/е/и/й/ь/у/ю),
  если термин длиннее 4 букв; иначе — целиком. Match = `основа + [а-яё]{0,3}` на
  границе слова. Примеры: «давление»->stem «давлени» -> давление/давления/давлением;
  «гард»->«гард» -> гард/гарда/гарде; «свип» -> свип/свипы/свипа.
- **Нерегулярности (беглые гласные и т.п.):** опциональное поле `stems?: string[]` у
  термина словаря переопределяет авто-основу. Пример: «Угол» -> `stems: ["угол","угл"]`
  (угол/угла/углом/угле).
- **Границы слова:** матч не внутри слова (перед — не кириллица/начало, основа + до 3
  доп. кириллических букв, после — не кириллица). «гард» не ловит «гардероб» (суффикс
  «ероб» > 3).
- **Многословные термины** («Закрытый гард», «Де-ла-Рива»): каждое слово по своей
  основе, разделители `\s+`/дефис как в термине. «закрытого гарда» ловится целиком.
- **Приоритет:** длинные термины раньше коротких (longest-first), чтобы «Закрытый
  гард» матчился целиком, а не как «гард». Перекрытия не допускаются.

## Архитектура

- `src/lib/bjj/glossaryMatch.ts` (чистый, TDD):
  - `buildMatchers(glossary, techniques)` -> отсортированный longest-first список
    целей `{ kind: "glossary"|"technique", key, regex, ... }` (мемо на модуль-уровне).
  - `findMatches(text: string, opts: { excludeTechId?: number }): Match[]` где
    `Match = { start, end, text, kind, ref }` (ref = term-объект или techId).
    Первое вхождение на термин в пределах ОДНОГО вызова (per-block).
- `src/components/bjj/GlossaryText.tsx`:
  - Проп `text` (plain) или `html` (строка с `<br>`/простыми тегами) + `excludeTechId`.
  - Безопасно проходит по ТЕКСТОВЫМ сегментам (html разбивается по тегам, матч только
    в тексте — теги не трогаем), оборачивает матчи в `<button>`/`<a>`.
  - Концепт -> кнопка, открывает поповер (термин + en + definition). Техника -> `Link`.
  - Поповер: один открытый на страницу, позиционируется у слова, закрытие по тапу
    вне/Esc. Позиция без `position:fixed` (в потоке/absolute относительно контейнера).
- Встройка в `src/routes/technique.$id.tsx`: `concept` через `GlossaryText text=...`,
  `mechanics`/`keyPoints` (сейчас через MechanismSection/InsightSection с html) —
  прогонять их html через `GlossaryText html=...` (проверить, что секции принимают
  готовый React вместо dangerouslySetInnerHTML, или обернуть).

## Не входит (YAGNI)

- Подсветка в `when`/`mistakes`/`drills` (можно позже).
- Полноценная лемматизация/словарь словоформ — эвристики основы достаточно.
- Тап-цели для терминов вне словаря и без карточки.

## Тест-план (glossaryMatch.test.ts)

- Склонения одиночных: гард/гарда/гарде; свип/свипы; давление/давления.
- Беглый гласный: угол/угла (через stems).
- Границы: «гард» НЕ в «гардероб»; матч в начале/конце строки, после запятой.
- Longest-first: «закрытый гард» целиком, не «гард».
- Первое вхождение: два «гарда» в тексте -> один матч.
- Техника vs концепт: название техники -> kind "technique"; исключение self-id.
- Многословный: «закрытого гарда» (оба слова склонены).
