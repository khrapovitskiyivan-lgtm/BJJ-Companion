# Спек: экран награды после записи дневника

Дата: 2026-07-18
Статус: согласован (превью в чате; рост стата — дробные проценты)

## Цель

Главный дофаминовый момент приложения: сразу после сохранения новой записи
дневника показать, что изменила именно эта тренировка. Модалка с 2-3 дельтами.
XP-экономика (спек геймификации, п.6) встроится в этот же экран следующей
итерацией — базовый экран не должен ей мешать.

## Слоты (максимум 3, минимум 1)

1. **План недели** — показывается всегда.
   - Частота задана: «Неделя: N из M» по неделе даты записи (запись задним
     числом оценивается в своей неделе). Квота добита именно этой записью:
     золотой акцент и серия недель в плане (planStreak). Тренировочный день
     сверх квоты: «Сверх плана» золотом. Вторая запись в тот же день: нейтрально.
   - Частоты нет: дневной стрик. Формула выносится из ActivityHeatmap в
     plan.ts::dayStreak, календарь переводится на неё (DRY).
2. **Рост стата** — топ-стат по дробной дельте до/после записи.
   «После» повторяет save(): отработка +1.5 веса, not_started -> in_progress.
   В StatScore добавляется pctExact (без округления), pct остаётся целым.
   Слот скрыт, если дельта нулевая (у техник записи нет тегов статов или кап 100%).
3. **Защита** — только если в записи есть техника, у которой в setup_from
   один из topCatchers дневника (2+ поимок, включая новую запись): «Закрываешь
   дыру». Приоритет: самый частый catcher; среди защит записи — побеги первыми
   (ранжирование как в defensesFor).

## Реализация

- `src/lib/bjj/reward.ts` — чистый computeEntryReward({entriesBefore, entry,
  progressBefore, techniques, frequency, today}). Детерминирован: «сегодня» и
  техники параметрами.
- `src/components/bjj/EntryReward.tsx` — EntryRewardSheet на общем Sheet:
  карточки-дельты каскадом (tw-animate-css + animationDelay), бар стата
  анимируется от pctBefore к pctAfter, подпись «+X,X%». Золото только у
  достижений (квота, сверх плана, серия); защита в тоне primary (прогресс).
- `src/routes/diary.tsx` — расчёт в save() до addEntry, только для новой
  записи (редактирование без награды). Рендер шторки по состоянию reward.

## Тесты

reward.test.ts: квота добита / не добита / сверх плана / вторая запись в день,
задним числом в прошлую неделю, дневной стрик без частоты, топ-стат и дробная
дельта, защита найдена / не найдена. plan.test.ts: dayStreak, weekDays.
stats.test.ts: pctExact дробный.

## Не входит

XP-экономика (следующая итерация, встраивается в этот экран), шеринг (очередь
п.18.7), звук и конфетти-библиотеки (каскад карточек чистым CSS).
