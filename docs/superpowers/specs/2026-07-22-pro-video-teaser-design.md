# PRO-видео тизер + сигнал спроса — дизайн

Дата: 2026-07-22. Статус: согласован, к реализации.

## Задача
Видео — топ-запрос живого опроса (п.25 CLAUDE.md), но собственная студийная съёмка
310 техник — большая работа и платный контент. Прежде чем вкладываться в съёмку,
нужно **измерить спрос**: сколько людей вообще хотят видео и **какие техники хотят
первыми**. Одновременно — посеять ценность PRO без реальной покупки (ставка РОСТ,
не монетизация; п.23).

Эта итерация = **тизер PRO-видео на карточке + сигнал спроса в телеметрии +
готовность пайплайна**. Показа реального видео/картинок нет (материала нет).

## Вне scope (осознанно отложено)
- Реальная покупка/тариф/entitlement/Telegram Stars, paywall-UI. Отдельная сессия
  после валидации спроса.
- Сама съёмка видео и хостинг.
- Картинки/скриншоты в бесплатной версии: стоп-кадр невозможен без отснятого
  материала, генерация дорогая и ненадёжная (решение пользователя) — не делаем.
- Отдельный UI-список «кто что запросил» для пользователя: спрос читается в
  телеметрии, не в интерфейсе.

## Решения (согласованы)
- **Где:** тизер на ВСЕХ техниках (310). Подмножество предрешало бы, что снимать,
  и смещало бы выборку. Блок визуально тихий; копирайт честный («скоро», не «есть»).
- **Апгрейд:** как только у техники появится `videoUrl` — карточка показывает
  реальный плеер (`VideoBlock`, уже готов) вместо тизера. Автоматически, без правок.
- **Кнопка:** «Мне интересно» → после тапа «Интерес учтён».
- **Что запоминаем:** событие спроса ПО ТЕХНИКЕ (`technique_id` в detail), не только
  счётчик — даёт ранжирование «что снимать первым». Дедуп: одно устройство = один
  интерес на технику (локальная отметка). Кнопка на этой технике потом показывает
  «Интерес учтён», повторно не тапается.
- **Место:** там же, где рендерится `VideoBlock` — после описания, перед «Когда
  применять» (technique.$id.tsx, ~строка 217).

## Компоненты (изолированные, одна ответственность)

### 1. `src/lib/bjj/videoInterest.ts` (модуль состояния интереса)
Что делает: хранит множество id техник, по которым пользователь тапнул «интересно»,
в localStorage; шлёт событие спроса один раз на технику.
Интерфейс:
- `hasVideoInterest(id: number): boolean` — тапал ли уже (SSR-safe: false на сервере).
- `markVideoInterest(id: number): void` — добавить в множество + `track('pro_video_interest', String(id))`; если уже в множестве — ничего (дедуп).
Зависимости: `localStorage`, `track` из telemetry.ts.
Ключ: `bjj.videoInterest.v1` (массив id в JSON).
Тестируемо: чистая логика над мок-localStorage (как остальные модули).

### 2. `src/components/bjj/technique/ProVideoTeaser.tsx` (UI-тизер)
Что делает: компактный блок-тизер платного видео с крючком интереса.
Пропс: `techniqueId: number`.
Состояния (по `hasVideoInterest` + локальный useState после тапа):
- **default:** компактный «заблокированный» кадр (play-кружок приглушённый) +
  золотой бейдж «Скоро в PRO»; заголовок «Видео-разбор приёма»; подпись
  «Снято в студии, с деталями под твоё телосложение. Мост от „понял“ к „сделал“.»;
  кнопка `Button` (variant primary, navy) «Мне интересно».
- **после тапа / уже интересно:** плашка «Интерес учтён» (в тоне success) +
  подпись «Поможет решить, что снять первым.» Кнопки нет.
Токены дизайн-системы: кнопка `--primary` (navy), бейдж PRO `--brand-gold` заливка /
`--brand-gold-ink` текст (золото). Кадр — уменьшенной высоты (не полный 16:9), чтобы
×310 не давил. Общие примитивы: `Button` из `components/bjj/ui.tsx`, иконки lucide
(Play, Lock, Check). Гидрация: `hasVideoInterest` читать после гидратации (не в
useState-инициализаторе; правило стора), первый рендер — default.
Приватность: `track` уже гейтится `hasConsent()` — в локальном режиме ничего не
уходит (осознанная плата за уважение к согласию; отказ серверно не измеряется).

### 3. `src/lib/bjj/telemetry.ts` (+1 событие)
Добавить `'pro_video_interest'` в union `TelemetryEvent`. Detail = `technique_id`
(строка, сервер режет до 32 симв. — id влезает). Без dailyDedup: дедуп обеспечивает
локальное множество в videoInterest.ts.

### 4. `src/routes/technique.$id.tsx` (место вставки)
Заменить `{videoUrl && <VideoBlock .../>}` на
`{videoUrl ? <VideoBlock .../> : <ProVideoTeaser techniqueId={tech.id} />}`.
Импорт ProVideoTeaser. Больше ничего.

### 5. `docs/sql/2026-07-22-video-interest-telemetry.sql` (применяет пользователь)
`create or replace` функции `bjj_track` с добавленным `'pro_video_interest'` в белый
список (образец docs/sql/2026-07-22-telemetry-partners.sql). В комментарии — запрос
чтения спроса:
`select detail as technique_id, count(*) as interest from bjj_events
 where event='pro_video_interest' group by detail order by interest desc;`
Событие начнёт капать со следующего дня после применения SQL (до этого RPC молча
отбрасывает незнакомое имя — не мешает приложению).

## Пайплайн видео (минимальная заготовка)
Чтобы «снял одну технику → загорелась» без доработки кода:
- `data/video-urls.json` — карта `{ "<id>": "<url>" }` (сейчас пустая `{}`).
- `scripts/build-data.mjs` — читает эту карту и проставляет `videoUrl` технике по id
  (если ключа нет — поле не задаётся). Убрать `as any` в technique.$id.tsx (тип
  `Technique.videoUrl?` уже есть).
CSV не трогаем (колонка, пустая у 310 строк, раздувала бы дифф; отдельная карта
проще для десятка ссылок). Заполнение карты реальными URL — будущий шаг.

## Данные, что уходит на сервер
Только `pro_video_interest` + `technique_id` + `device_id` (как вся телеметрия).
Ни содержимого, ни персональных данных. Гейт согласия соблюдён.

## Тестирование
- vitest: `videoInterest.ts` — has/mark, идемпотентность (второй mark не шлёт),
  персистентность множества, SSR-safe (false без window). Мок localStorage как в
  существующих тестах.
- Рантайм (preview): открыть технику → виден тизер; тап «Мне интересно» → «Интерес
  учтён»; перезагрузка → состояние «учтено» держится; перехват fetch → ровно один
  `bjj_track` с event=pro_video_interest и detail=id, повторный тап не шлёт; техника
  со стаб-`videoUrl` в карте → показывает `VideoBlock`, не тизер. Обе темы.

## Критерии готовности
- Тизер виден на карточке любой техники без видео; апгрейд на плеер при наличии URL.
- Тап шлёт ровно одно событие спроса по технике, дедуп держит, состояние
  персистентно локально.
- Сборка данных зелёная, vitest зелёный, рантайм-проверка пройдена в обеих темах.
- SQL-файл готов к применению; без него приложение работает (события глотаются).
