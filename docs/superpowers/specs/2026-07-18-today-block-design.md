# Стартовый экран: блок «Сегодня» + переименование генератора в «Отработку» (п.18.9)

Дата: 2026-07-18. Решения пользователя: вариант «блок Сегодня» (старт остаётся /progress);
терминологию развести — «тренировка» только про клубные занятия (дневник, план, напоминания),
«отработка» — всё, что генерирует приложение; раздел генератора переименовать во всём приложении.
Кнопка в блоке одна и всегда про дневник — путаницы с генератором нет по построению.

## 1. Блок «Сегодня»

Место: `src/routes/progress.tsx`, между PageHeader и hero-сеткой. Ниже вся страница без изменений.

Модель — чистая функция в новом `src/lib/bjj/todayCard.ts` (+ тесты, по образцу reward.ts):
`todayCardModel(entries, frequency, today)` возвращает состояние блока:

- `loggedToday` — есть ли запись с датой сегодня.
- При заданной частоте: `done`/`quota` недели (из `weekStatus`), число сверхплановых,
  `daysLeft` — сколько тренировочных дней осталось добить квоту: модель недели как у бота
  (Пн-Сб, вс выходной), сегодня считается, если записи ещё нет. Вынести хелпером
  `daysLeftInWeek(today, loggedToday)` в `plan.ts` с тестами (пн, сб, вс, с записью/без).
- `planStreak` (если >= 2 — показываем серию недель), без частоты — `dayStreak`.

Вид (компонент `TodayCard.tsx`, тонкий, всё из модели):

- Кикер «Сегодня» + дата справа (`Intl.DateTimeFormat("ru")`, «чт, 18 июля»).
- Записи сегодня нет: рамка primary (выделение), строка «На этой неделе N из Q»,
  сегменты квоты (заливка `--status-progress`, сверхплановые `--brand-gold`, пустые muted),
  подстрока «До плана X, осталось Y дней» (при закрытой квоте — «План недели закрыт»,
  сверхплановые — золотом «+K сверх плана»), кнопка primary во всю ширину
  «Записать в дневник» -> `/diary?add=1`.
- Запись есть: рамка обычная, вместо даты зелёная галочка «записано», кнопки НЕТ.
- Серия: «N недель в плане подряд» золотым (`--brand-gold-ink`) с Flame — только при planStreak >= 2.
- Без частоты: вместо строки недели — дневной стрик «N дней подряд» (dayStreak > 0), иначе
  только кикер + кнопка.
- Рендер после `hydrated` (SSR-safe, как остальной /progress), до — null.

`/diary?add=1`: search-параметр в `diary.tsx` (validateSearch, паттерн `?s=` из situations),
на входе с ним `adding` стартует true. Телеметрию для кнопки не добавляем (нового события
нет в белом списке SQL; section_open дневника сработает сам; добавить можно позже).
